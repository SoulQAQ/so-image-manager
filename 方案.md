# 本地AI图片管理系统（Android）技术方案 v1.2

> 更新时间：2026-05-11  
> 本版本重点：确定 UI 技术栈为 `原生 Compose + WebView(Vue3 + Vant4)`，并将 embedding/多模态重排/人脸聚类/去重/近重复治理后移到 v3/v4+

## 一、项目目标

构建一个**本地优先**的智能图片管理 App（Android），实现：

- 自动识别新增图片
- AI 生成图片描述与多标签分类
- 支持多维标签检索与全文模糊搜索（FTS5）
- 支持人工修正 AI 结果
- 本地 SQLite 持久化存储

---

## 三、整体架构

```text
Android App
├── 图片接入层（MediaStore 扫描 + Share Intent + Photo Picker）
├── 预处理层（压缩、旋转纠正、sha256）
├── AI 分析层（HTTP API，结构化 JSON 输出）
├── 标签标准化层（canonical + alias）
├── 存储层（SQLite + Room）
├── 搜索层（FTS5 + 规则粗排）
├── 原生 UI 层（Jetpack Compose）
└── H5 UI 层（WebView + Vue3 + Vant4）
```

原则：

- 图片文件不移动、不重命名
- 文件与元数据解耦，组织结构由数据库表达
- AI 结果与用户结果双存，可回滚

## 四、App UI 技术栈方案（新增模块）

## 4.1 选型结论

- 原生层：`Jetpack Compose + Material3 + Navigation Compose`
- H5 层：`WebView + Vue3 + Vant4`
- 状态管理：
  - 原生：`ViewModel + StateFlow`
  - H5：`Pinia`

说明：

- 高性能、重交互、系统能力密集页面优先 Compose
- 业务配置、表单、活动页、轻交互页面可放 H5

## 4.2 建议页面分工

Compose 页面（建议）：

- 首页图库流
- 大图浏览页
- 搜索入口与结果骨架
- 系统级权限引导页

WebView 页面（建议）：

- 标签管理后台（alias/canonical 维护）
- 筛选器高级面板
- 帮助/设置/规则说明页

## 4.3 H5 打包与发布策略

- Vue 工程使用 `Vite + Vue3 + Vant4`
- `pnpm build` 后产物放入 `app/src/main/assets/h5/`
- WebView 默认加载：`file:///android_asset/h5/index.html`
- 首期不做远程热更新（先随 App 版本发布，降低复杂度）

## 4.4 Native <-> H5 通信（JSBridge）

统一 JSON 协议，建议只保留少量白名单方法：

- `app.getDeviceInfo`
- `app.pickImage`
- `app.queryImageTags`
- `app.saveUserTags`
- `app.openImageDetail`

返回统一结构：

```json
{
  "code": 0,
  "message": "ok",
  "data": {}
}
```

## 4.5 WebView 安全基线

- 默认只允许加载本地 assets 或白名单域名
- `addJavascriptInterface` 仅暴露必要对象与方法
- 禁止任意 file URL 跨域访问
- Release 关闭 WebView 调试
- 所有 Bridge 请求做参数校验与来源校验

## 五、核心流程设计

### 1. 新增图片自动处理

```text
新增图片 -> 入队 -> 预处理(压缩/旋转/sha256)
-> AI 分析 -> 标签标准化 -> 写入 DB -> 更新 FTS
```

### 2. 看图查标签（库内）

```text
打开图片 -> 查询 image/image_ai/image_tag
-> 展示 AI 与用户标签
-> 人工修正 -> 写回 user_* 并刷新索引
```

### 3. 看图查标签（外部临时）

```text
Photo Picker/分享进入单图
-> 计算 sha256 查缓存
-> 命中缓存则直接返回
-> 未命中则调用 AI 分析
-> 返回结果（可选一键入库）
```

### 4. 以图反查（v2）

```text
输入图 -> 生成 query tags + hash
-> 标签召回候选 -> hash + 标签重合度粗排
-> 输出 TopK
```

## 六、AI 输出结构（Schema v1.2）

```json
{
  "caption": "一个人在海边散步",
  "categories": ["真实照片/风景", "真实照片/人像"],
  "tags": [
    {"name": "海边", "score": 0.95},
    {"name": "沙滩", "score": 0.93},
    {"name": "夕阳", "score": 0.87}
  ],
  "search_tokens": ["海边", "沙滩", "人像", "夕阳", "墨镜"],
  "ocr": [],
  "safety": {"nsfw": false},
  "score": {"quality": 0.92}

```

约束：

- categories 仅可从预定义分类树选择
- tags 可自由生成，但入库前必须 canonical 化
- `schema_version`、`prompt_version`、`ai_model` 必须落库

## 七、数据库设计（SQLite + Room）

### 1. image

```sql
CREATE TABLE image (
    id INTEGER PRIMARY KEY AUTOINCREMENT,

    uri TEXT NOT NULL,
    path TEXT,
    sha256 TEXT NOT NULL UNIQUE,

    width INTEGER,
    height INTEGER,
    mime_type TEXT,
    file_size INTEGER,

    create_time INTEGER,
    modify_time INTEGER,
    import_time INTEGER,

    ai_status INTEGER DEFAULT 0,

    ai_model TEXT,
    prompt_version TEXT,
    schema_version TEXT,

    analyzed_at INTEGER,
    user_modified INTEGER DEFAULT 0
);
```

### 2. image_ai

```sql
CREATE TABLE image_ai (
    image_id INTEGER PRIMARY KEY,

    ai_caption TEXT,
    ai_tags_json TEXT,
    ai_categories_json TEXT,
    ai_search_tokens TEXT,

    user_caption TEXT,
    user_tags_json TEXT,
    user_categories_json TEXT,

    ocr_text TEXT,
    raw_json TEXT,

    FOREIGN KEY(image_id) REFERENCES image(id)
);
```

### 3. tag / image_tag / tag_alias

```sql
CREATE TABLE tag (
    id INTEGER PRIMARY KEY,
    name TEXT UNIQUE
);

CREATE TABLE image_tag (
    image_id INTEGER,
    tag_id INTEGER,
    confidence REAL,
    PRIMARY KEY(image_id, tag_id)
);

CREATE TABLE tag_alias (
    alias TEXT PRIMARY KEY,
    canonical TEXT NOT NULL
);
```

### 4. image_fts

```sql
CREATE VIRTUAL TABLE image_fts USING fts5(
    image_id UNINDEXED,
    content
);
```

### 5. image_query_cache（临时看图缓存）

```sql
CREATE TABLE image_query_cache (
    sha256 TEXT PRIMARY KEY,
    ai_model TEXT NOT NULL,
    schema_version TEXT NOT NULL,
    result_json TEXT NOT NULL,
    created_at INTEGER NOT NULL,
    expire_at INTEGER NOT NULL
);
```

### 6. image_feature（v2 使用）

```sql
CREATE TABLE image_feature (
    image_id INTEGER PRIMARY KEY,
    phash TEXT,
    updated_at INTEGER,
    FOREIGN KEY(image_id) REFERENCES image(id)
);
```

## 八、搜索策略

### 1. 文本检索（主链路）

```text
用户输入 -> canonical 化 -> FTS5 MATCH -> image_id -> JOIN image
```

示例：

```sql
SELECT image_id
FROM image_fts
WHERE image_fts MATCH '海边 墨镜'
ORDER BY bm25(image_fts)
LIMIT 200;
```

### 2. 看图查标签

- 先查缓存（sha256 + model + schema）
- 失效后调用 AI
- 返回可“临时查看”或“保存入库”

### 3. 以图反查（v2）

- 召回：query tags 命中候选
- 重排：标签重合度 + hash 距离
- 解释：返回共同标签和相似原因

---

## 九、中文搜索策略

结论：SQLite FTS5 默认不做中文自动分词。

v1/v2 方案：

- 依赖 AI 输出 `search_tokens`
- canonical 化后再入 FTS

后续可选：

- jieba 分词
- trigram tokenizer（需验证当前 Android SQLite 驱动能力）

## 十一、关键风险与缓解

- AI 误标：保留用户修正结果，支持重跑
- 模型升级漂移：强制记录 model/prompt/schema 版本
- WebView 安全：白名单 + 最小 Bridge 面 + 参数校验
- 队列堆积：幂等 + 退避重试 + 可恢复
- FTS 体积增长：周期性优化与分批重建
