# 项目总览

本文用于让后续 Agent 快速理解当前仓库结构、每个文件夹用途与关键文件职责。

## 1. 根目录结构

- `app/`
  - Android 主工程（Kotlin + Jetpack Compose），APK 构建核心模块。
- `h5/`
  - H5 子工程（Vue3 + Vite + Vant），构建后输出到 `app/src/main/assets/h5/` 供 WebView 使用。
- `apk/`
  - 已发布安装包与版本记录。
- `env/`
  - 本地环境配置（如 API Key）。
- `gradle/`
  - Gradle Wrapper 与版本目录配置。
- `docs/`
  - 项目文档目录（本文件所在位置）。
- `.gradle/`、`.kotlin/`、`build/`
  - 构建缓存与输出目录，自动生成。
- `.git/`
  - Git 元数据。
- `.claude/`
  - 本地代理相关目录（不属于业务代码）。

## 2. 根目录关键文件

- `AGENTS.md`
  - 项目级开发约束与风格要求（UI 风格、协作规则等）。
- `README.md`
  - 项目说明、技术栈与基础启动指引。
- `CLAUDE.md`
  - 给代理的项目操作说明（命令、架构、路径、注意事项）。
- `build.gradle.kts`
  - 根 Gradle 配置。
- `settings.gradle.kts`
  - Gradle 模块声明（包含 `:app`）。
- `gradle.properties`
  - Gradle/Kotlin 全局构建参数。
- `local.properties`
  - 本机 Android SDK 路径与本地构建参数（如 `aiApiKey`）。
- `logo.png`
  - 应用图标源文件（用于生成 mipmap 资源）。
- `开始清单.md`、`第二阶段清单.md`、`方案.md`
  - 规划文档与阶段任务说明。

## 3. Android 模块（`app/`）

### 3.1 构建与配置

- `app/build.gradle.kts`
  - Android 模块构建脚本。
  - 定义 `versionCode/versionName`、签名配置、BuildConfig 注入（`AI_API_URL`、`AI_API_KEY`、`AI_MODEL`）。
- `app/proguard-rules.pro`
  - 混淆规则（当前 release 未开启混淆）。

### 3.2 Android 资源

- `app/src/main/AndroidManifest.xml`
  - 应用声明、权限、入口 Activity。
- `app/src/main/res/values/`
  - `strings.xml`（文案）、`colors.xml`（基础色）、`themes.xml`（主题映射）。
- `app/src/main/res/mipmap-*/`
  - 各分辨率应用图标位图（含 `ic_launcher_v2*`）。
- `app/src/main/res/mipmap-anydpi-v26/`
  - 自适应图标定义（`ic_launcher.xml`、`ic_launcher_round.xml`）。
- `app/src/main/res/drawable/ic_launcher_foreground.xml`
  - 图标前景层定义。

### 3.3 Native 业务代码（`app/src/main/java/cn/soul2/imageai/`）

#### 入口与导航

- `MainActivity.kt`
  - 应用入口，加载 Compose 根 UI。
- `AppNavigation.kt`
  - Compose 导航图，定义首页、设置、WebView、图片详情等路由跳转。
- `ui/navigation/NavRoutes.kt`
  - 路由常量与带参数路由构造函数。

#### UI 层

- `ui/screens/GalleryScreen.kt`
  - 当前首页核心：
    - 顶部搜索框（在 ImageAI + 设置图标下方）
    - 按图集/按图片导入
    - 图片网格展示
    - 长按多选与 AI 标注入口
    - 图集列表读取与导入逻辑
- `ui/screens/ImageDetailScreen.kt`
  - 图片详情页：大图查看、标注状态、模型描述、标签展示。
- `ui/screens/SettingsScreen.kt`
  - 设置页（主题模式等）。
- `ui/screens/WebViewScreen.kt`
  - WebView 容器页，加载本地 H5。

#### 主题与设置

- `ui/theme/Color.kt`
  - 主题色定义（浅色/深色）。
- `ui/theme/Theme.kt`
  - MaterialTheme 组装逻辑。
- `ui/theme/Typography.kt`
  - 字体样式。
- `ui/settings/ThemeSettings.kt`
  - 主题模式持久化（DataStore）。

#### AI 与网络

- `data/api/AiHttpClient.kt`
  - OkHttp 客户端与鉴权头处理。
- `data/api/AiService.kt`
  - AI 请求封装（OpenAI 兼容格式）。
- `data/api/model/AiAnalysisResult.kt`
  - AI 结构化返回模型（caption/tags/categories/search_tokens 等）。

#### 数据库（Room）

- `data/db/AppDatabase.kt`
  - Room 数据库入口与 FTS 初始化。
- `data/db/entity/*.kt`
  - 数据实体定义（Image、ImageAi、Tag、TagAlias、ImageTag、ImageQueryCache、ImageFeature）。
- `data/db/dao/*.kt`
  - DAO 接口（增删改查、搜索、关联关系）。

#### 领域服务

- `domain/service/ImagePreprocessor.kt`
  - 图片预处理：压缩、EXIF、SHA256。
- `domain/service/ImageScanner.kt`
  - 图片扫描相关能力（历史逻辑，现已弱化全库扫描使用）。
- `domain/service/ImageProcessQueue.kt`
  - 图片处理队列。
- `domain/service/AiResultProcessor.kt`
  - AI 结果落库与结构化处理。
- `domain/service/SearchService.kt`
  - 搜索相关服务。

#### JS Bridge

- `webview/JsBridge.kt`
  - Native 与 H5 通信桥。

### 3.4 H5 构建产物（供 WebView 使用）

- `app/src/main/assets/h5/`
  - H5 构建输出目录（`index.html` + `assets/*`）。
  - 由 `h5` 工程构建后覆盖生成。

## 4. H5 子工程（`h5/`）

### 4.1 构建与依赖

- `h5/package.json`
  - 前端依赖与脚本（`dev`、`build`、`build:android`）。
- `h5/vite.config.ts`
  - Vite 配置，`build:android` 输出到 Android assets。
- `h5/tsconfig*.json`
  - TypeScript 配置。

### 4.2 源码

- `h5/src/main.ts`
  - Vue 应用入口。
- `h5/src/App.vue`
  - 根组件。
- `h5/src/router/index.ts`
  - 路由定义。
- `h5/src/types/index.ts`
  - 类型定义（含桥接类型）。
- `h5/src/views/`
  - `HomeView.vue`：H5 首页
  - `SearchView.vue`：搜索页
  - `ImageDetailView.vue`：图片详情页
  - `TagManagerView.vue`：标签管理页

## 5. 发布与版本文件（`apk/`）

- `apk/image-ai-v*.apk`
  - 历史发布包。
- `apk/current_version_is_xxx`
  - 当前版本标识文件（仅保留最新版本）。
- `apk/ver_change_log.md`
  - 版本变更日志（每次打包需更新）。

## 6. 环境配置（`env/`）

- `env/apikey.txt`
  - 本地 API Key 源文件。
  - 构建时由 `app/build.gradle.kts` 读取并注入 BuildConfig。

## 7. 构建命令速查

### Android

- `./gradlew assembleDebug`
- `./gradlew assembleRelease`
- `./gradlew clean`

Windows 可用：
- `gradlew.bat assembleRelease`

### H5

- `cd h5 && pnpm install`
- `cd h5 && pnpm build:android`

## 8. 当前架构状态（给后续 Agent 的重点）

1. 主入口是原生 Compose（GalleryScreen），不是纯 WebView。  
2. WebView 仍保留，作为补充管理界面。  
3. AI 调用走 OpenAI 兼容接口，核心配置来自 BuildConfig。  
4. 图片导入以“用户主动选择”为主（图集/图片），避免默认全库托管。  
5. 每次发布必须同步更新 `apk/current_version_is_xxx` 与 `apk/ver_change_log.md`。
