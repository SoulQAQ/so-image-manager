# 开始清单（Phase 1）

> 目标读者：执行开发任务的 Agent  
> 目标版本：v1 起步版  
> 最终目标：创建一个可运行的 Android App 工程（Compose + WebView + Vue3 + Vant4），并可正常打包 APK

## 1. 第一阶段目标（必须全部满足）

- [ ] 成功创建 Android 工程（Kotlin + Compose）
- [ ] App 首页可启动并正常显示 Compose 页面
- [ ] App 内可打开 WebView 页面
- [ ] WebView 能加载本地 H5（Vue3 + Vant4）页面
- [ ] Native 与 H5 至少打通 1 个 JSBridge 方法（如 `ping`）
- [ ] `assembleDebug` 成功并产出 APK
- [ ] `assembleRelease` 成功并产出 release APK（可为未签名包）

## 2. 非目标（本阶段不要做）

- [ ] 不实现 embedding 语义检索
- [ ] 不实现多模态重排
- [ ] 不实现人脸聚类、去重、近重复治理
- [ ] 不实现远程热更新
- [ ] 不做复杂业务逻辑（AI/DB 先留接口或占位）

## 3. 环境准备清单

- [ ] JDK 17
- [ ] Android SDK Platform 36（或本机 Android Studio 稳定通道可用最高）
- [ ] Android Build Tools 36.0.0
- [ ] Android Gradle Plugin 9.x 稳定版
- [ ] Gradle 9.1+
- [ ] Node.js LTS（建议 20+）
- [ ] pnpm（建议 9+）

建议先执行环境检查命令：

```powershell
java -version
node -v
pnpm -v
```

## 4. 任务分解（按顺序执行）

### T1. 初始化 Android 工程

- [ ] 使用 Android Studio 新建 `Empty Activity`（Compose）工程
- [ ] 语言：Kotlin
- [ ] `minSdk` 建议 26（最低不低于 24）
- [ ] 包名按团队规范（例如 `com.xxx.aiphoto`）

完成标准：

- [ ] 可执行 `./gradlew tasks`
- [ ] 工程可在模拟器/真机启动

### T2. 搭建 Compose 基础壳

- [ ] 创建 `NavHost`，至少包含两个路由：`home`、`webview`
- [ ] `home` 页面至少有一个按钮跳转到 `webview`
- [ ] 统一 Material3 主题（浅色先行）

完成标准：

- [ ] App 启动默认进入 `home`
- [ ] 点击按钮可稳定进入 `webview`

### T3. 接入 WebView 容器页

- [ ] 在 Compose 中通过 `AndroidView` 承载 `WebView`
- [ ] 使用 `WebViewAssetLoader` 加载本地资源
- [ ] 加载地址使用：`https://appassets.androidplatform.net/assets/h5/index.html`

完成标准：

- [ ] WebView 页面可正常打开本地内容
- [ ] 页面刷新/切后台恢复无崩溃

### T4. 落实 WebView 安全基线（必须）

- [ ] 仅允许本地 assets 或白名单域名
- [ ] `setAllowFileAccess(false)`
- [ ] `setAllowContentAccess(false)`
- [ ] `setAllowFileAccessFromFileURLs(false)`
- [ ] `setAllowUniversalAccessFromFileURLs(false)`
- [ ] Release 关闭 WebView 调试

完成标准：

- [ ] 安全设置在代码中可见
- [ ] 不使用 `file://` 直接加载本地页

### T5. 创建 H5 子工程（Vue3 + Vant4）

- [ ] 在仓库创建 `h5/` 目录
- [ ] 初始化 `Vite + Vue3` 项目
- [ ] 安装 `vant@4`（使用最新稳定 patch）
- [ ] 页面至少包含：`NavBar`、`Button`、`Cell` 各 1 个

建议命令：

```powershell
# 在项目根目录
pnpm create vite h5 --template vue-ts
cd h5
pnpm add vant
pnpm add -D @types/node
pnpm install
```

完成标准：

- [ ] `pnpm build` 成功
- [ ] 能看到包含 Vant 组件的静态页面

### T6. H5 构建产物接入 Android assets

- [ ] 配置 Vite `base: './'`
- [ ] 将 H5 构建产物输出到 `app/src/main/assets/h5/`
- [ ] 提供脚本一键执行（如 `pnpm build:android`）

完成标准：

- [ ] 每次执行脚本后，Android assets 内有最新 H5 文件
- [ ] App 内 WebView 打开的是最新构建内容

### T7. 打通最小 JSBridge

- [ ] Native 暴露 `AppBridge`（例如 `ping(message)`）
- [ ] H5 可调用并展示返回结果
- [ ] 约定统一返回结构：`{ code, message, data }`

完成标准：

- [ ] 点击 H5 按钮可触发 Native 方法
- [ ] H5 界面能看到回传结果

### T8. 工程脚本与文档

- [ ] 根目录补充 `README.md`（含运行、打包步骤）
- [ ] 写明 H5 构建命令与 Android 打包命令
- [ ] 写明产物路径与常见问题排查

完成标准：

- [ ] 新成员按 README 可在新环境构建成功

### T9. 打包验证（关键里程碑）

执行：

```powershell
# 项目根目录
./gradlew clean assembleDebug
./gradlew assembleRelease
```

完成标准：

- [ ] Debug APK 存在：`app/build/outputs/apk/debug/app-debug.apk`
- [ ] Release APK 存在：`app/build/outputs/apk/release/`
- [ ] 构建日志无阻断错误（Exit Code = 0）

### T10. 冒烟测试

- [ ] 安装 Debug APK 到模拟器/真机
- [ ] 启动后能进入 Home
- [ ] 能打开 WebView 页面
- [ ] Vant 组件正常渲染
- [ ] JSBridge 调用成功

## 5. 交付物清单（Phase 1 结束时）

- [ ] Android 工程源码（Compose）
- [ ] H5 工程源码（Vue3 + Vant4）
- [ ] WebView 容器与安全配置代码
- [ ] 最小 JSBridge 实现
- [ ] 可执行的打包命令
- [ ] Debug/Release APK 构建产物
- [ ] README 使用说明

## 6. 一票否决项（任一不满足即不通过）

- [ ] 不能产出 APK
- [ ] App 无法启动到首页
- [ ] WebView 无法加载本地 H5
- [ ] H5 不能使用 Vant 组件
- [ ] 没有最小 JSBridge 验证闭环

## 7. 建议的提交节奏（可选但推荐）

- [ ] Commit 1：初始化 Android Compose 工程
- [ ] Commit 2：WebView 容器 + 安全基线
- [ ] Commit 3：H5(Vue3 + Vant4) + assets 接入
- [ ] Commit 4：JSBridge 打通 + README + 打包验证

## 8. 里程碑完成判定（最终）

当以下条件全部满足，Phase 1 视为完成：

- [ ] `./gradlew clean assembleDebug` 成功
- [ ] `./gradlew assembleRelease` 成功
- [ ] 产物路径下存在 APK
- [ ] 实机/模拟器冒烟测试全部通过
