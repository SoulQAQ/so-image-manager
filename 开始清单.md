# 开始清单（Phase 1）

> 目标读者：执行开发任务的 Agent  
> 目标版本：v1 起步版  
> 最终目标：创建一个可运行的 Android App 工程（Compose + WebView + Vue3 + Vant4），并可正常打包 APK

## 1. 第一阶段目标（必须全部满足）

- [x] 成功创建 Android 工程（Kotlin + Compose）
- [x] App 首页可启动并正常显示 Compose 页面
- [x] App 内可打开 WebView 页面
- [x] WebView 能加载本地 H5（Vue3 + Vant4）页面
- [x] Native 与 H5 至少打通 1 个 JSBridge 方法（如 `ping`）
- [x] `assembleDebug` 成功并产出 APK
- [x] `assembleRelease` 成功并产出 release APK（可为未签名包）

## 2. 非目标（本阶段不要做）

- [ ] 不实现 embedding 语义检索
- [ ] 不实现多模态重排
- [ ] 不实现人脸聚类、去重、近重复治理
- [ ] 不实现远程热更新
- [ ] 不做复杂业务逻辑（AI/DB 先留接口或占位）

## 3. 环境准备清单

- [x] JDK 17
- [x] Android SDK Platform 36（或本机 Android Studio 稳定通道可用最高）
- [x] Android Build Tools 36.0.0
- [x] Android Gradle Plugin 8.7.3
- [x] Gradle 8.9+
- [x] Node.js LTS（建议 20+）
- [x] pnpm（建议 9+）

建议先执行环境检查命令：

```powershell
java -version
node -v
pnpm -v
```

## 4. 任务分解（按顺序执行）

### T1. 初始化 Android 工程

- [x] 使用 Android Studio 新建 `Empty Activity`（Compose）工程
- [x] 语言：Kotlin
- [x] `minSdk` 建议 26（最低不低于 24）
- [x] 包名按团队规范（例如 `com.xxx.aiphoto`）

完成标准：

- [x] 可执行 `./gradlew tasks`
- [x] 工程可在模拟器/真机启动

### T2. 搭建 Compose 基础壳

- [x] 创建 `NavHost`，至少包含两个路由：`home`、`webview`
- [x] `home` 页面至少有一个按钮跳转到 `webview`
- [x] 统一 Material3 主题（浅色先行）

完成标准：

- [x] App 启动默认进入 `home`
- [x] 点击按钮可稳定进入 `webview`

### T3. 接入 WebView 容器页

- [x] 在 Compose 中通过 `AndroidView` 承载 `WebView`
- [x] 使用 `WebViewAssetLoader` 加载本地资源
- [x] 加载地址使用：`https://appassets.androidplatform.net/assets/h5/index.html`

完成标准：

- [x] WebView 页面可正常打开本地内容
- [x] 页面刷新/切后台恢复无崩溃

### T4. 落实 WebView 安全基线（必须）

- [x] 仅允许本地 assets 或白名单域名
- [x] `setAllowFileAccess(false)`
- [x] `setAllowContentAccess(false)`
- [x] `setAllowFileAccessFromFileURLs(false)`
- [x] `setAllowUniversalAccessFromFileURLs(false)`
- [x] Release 关闭 WebView 调试

完成标准：

- [x] 安全设置在代码中可见
- [x] 不使用 `file://` 直接加载本地页

### T5. 创建 H5 子工程（Vue3 + Vant4）

- [x] 在仓库创建 `h5/` 目录
- [x] 初始化 `Vite + Vue3` 项目
- [x] 安装 `vant@4`（使用最新稳定 patch）
- [x] 页面至少包含：`NavBar`、`Button`、`Cell` 各 1 个

建议命令：

```powershell
# 在项目根目录
pnpm create vite h5 --template vue-ts
cd h5
pnpm add vant
pnpm add -D @types/node
pnpm install
```

完成标准：

- [x] `pnpm build` 成功
- [x] 能看到包含 Vant 组件的静态页面

### T6. H5 构建产物接入 Android assets

- [x] 配置 Vite `base: './'`
- [x] 将 H5 构建产物输出到 `app/src/main/assets/h5/`
- [x] 提供脚本一键执行（如 `pnpm build:android`）

完成标准：

- [x] 每次执行脚本后，Android assets 内有最新 H5 文件
- [x] App 内 WebView 打开的是最新构建内容

### T7. 打通最小 JSBridge

- [x] Native 暴露 `AppBridge`（例如 `ping(message)`）
- [x] H5 可调用并展示返回结果
- [x] 约定统一返回结构：`{ code, message, data }`

完成标准：

- [x] 点击 H5 按钮可触发 Native 方法
- [x] H5 界面能看到回传结果

### T8. 工程脚本与文档

- [x] 根目录补充 `README.md`（含运行、打包步骤）
- [x] 写明 H5 构建命令与 Android 打包命令
- [x] 写明产物路径与常见问题排查

完成标准：

- [x] 新成员按 README 可在新环境构建成功

### T9. 打包验证（关键里程碑）

执行：

```powershell
# 项目根目录
./gradlew clean assembleDebug
./gradlew assembleRelease
```

完成标准：

- [x] Debug APK 存在：`app/build/outputs/apk/debug/app-debug.apk`
- [x] Release APK 存在：`app/build/outputs/apk/release/`
- [x] 构建日志无阻断错误（Exit Code = 0）

### T10. 冒烟测试

- [ ] 安装 Debug APK 到模拟器/真机
- [ ] 启动后能进入 Home
- [ ] 能打开 WebView 页面
- [ ] Vant 组件正常渲染
- [ ] JSBridge 调用成功
